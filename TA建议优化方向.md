# TA 建议优化方向

> 整理日期：2026-04-15

---

## TA 原始建议要点

### 一、训练方法优化

**1. LoRA → DoRA（优先尝试，无需改 pipeline）**
- TA 认为 DoRA 在性能上比 LoRA 更优
- 当前 LoRA 实验已经很充分，可以作为 **LoRA-based baseline**
- 先试 DoRA，看是否比 LoRA 好；如果好，再接着搞数据
- 甚至可以尝试**全量微调**（1.5B 模型很小，不用 LoRA 也能跑通）
- 最终选一个（LoRA / DoRA / 全量微调）作为后续训练基准

**2. 实现 Iterative DPO**
- 利用 SFT 模型的输出作为负样本
- DPO 学习中不仅需要好的结果，还需要差的结果来增强模型的 ranking 能力

### 二、数据层面优化

**3. SFT 数据：分阶段训练，不要直接混合**
- 先用 NuminaMath 进行基础逻辑对齐
- 再引入 Magpie-Reasoning 增强通用推理
- 小模型容易在混合训练中产生"灾难性遗忘"或逻辑混淆

**4. 数据多样性 > 数据量**
- 对 1.5B 小模型，"数据多样性"比"数据量"更重要
- 找个大模型合成更多、更丰富的数据作为补充，能很快看到结果

**5. DPO 数据质量 > 数量**
- 高质量的几千条数据比低质量的几万条更有意义
- 可以写脚本让大模型检查数据质量
- 用 Teacher-Guided 方法对 DPO 数据集进行清洗

**6. 增加 Long-CoT 合成数据**
- 当前数据集（如 Orca-Math）对推理步骤的颗粒度要求不够
- 用 Qwen2.5-72B 对 NuminaMath 中的难题生成更详尽的 Step-by-step 思考过程
- 确保 1.5B 学到的是"思维链条"而非"答案模板"

## 执行步骤（按 TA 建议的优先级）

### Step 1：DoRA 对比实验（最快出结果）
- [ ] 将 LoRA 替换为 DoRA，其他参数不变，直接跑一轮
- [ ] 对比 LoRA vs DoRA 的 GSM8K / BBH 结果
- [ ] 考虑是否尝试全量微调作为第三组对比
- [ ] 确定最终训练基准（LoRA / DoRA / 全量微调）

### Step 2：数据质量清洗与提升
- [ ] 写脚本让大模型检查现有 DPO 数据质量，过滤低质量样本
- [ ] 用 Teacher-Guided 方法重新构造 DPO 数据（SFT 模型错误输出 → 负样本，大模型正确输出 → 正样本）
- [ ] 用大模型合成更丰富的补充数据

### Step 3：SFT 分阶段训练
- [ ] 第一阶段：仅用 NuminaMath 训练（基础逻辑对齐）
- [ ] 第二阶段：引入 Magpie-Reasoning（通用推理增强）
- [ ] 对比分阶段 vs 直接混合的效果

### Step 4：Long-CoT 数据合成
- [ ] 用 Qwen2.5-72B 对 NuminaMath 难题生成详细推理链
- [ ] 将合成的 Long-CoT 数据加入 SFT 训练集

### Step 5：Iterative DPO
- [ ] 用 SFT 模型在训练集上推理，收集错误输出作为负样本
- [ ] 构造高质量 DPO pairs，重跑 DPO
- [ ] 可迭代多轮，逐步提升
