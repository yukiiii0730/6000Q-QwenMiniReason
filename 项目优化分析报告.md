## 三、创新性扩展方案

> 项目原始定位：通过大模型蒸馏增强小模型在特定领域的能力。  
> 当前方案的局限：大模型只隐式体现在已有数据集中，未真正实现动态蒸馏。

---

### 方向一：Teacher-Guided 合成 DPO 数据（最推荐，直接解决 DPO 失败）

**核心思路：** 用 Qwen2.5-7B（已有 API）在线生成 chosen/rejected 对，替代 Orca-Math 预制数据集

```
同一道数学题
  ├── Teacher (7B) 生成正确推理链  →  chosen
  └── Student (1.5B) 当前错误输出  →  rejected
```

**为什么比现有方案更好：**
- Orca-Math pairs 的数据分布可能与 1.5B 的能力边界不匹配（解释 DPO 失败的原因）
- Teacher-generated chosen 与题目分布完全一致
- rejected 来自 Student 的真实错误，是「最难的负样本」，DPO 信号更强

**实现伪代码：**
```python
for problem in math_problems:
    student_answer = student_model.generate(problem)
    if is_wrong(student_answer):
        teacher_answer = teacher_api.generate(problem)  # 复用已有 API 接口
        dpo_pairs.append({
            "prompt": problem,
            "chosen": teacher_answer,
            "rejected": student_answer
        })
```

**可行性：** 复用 `eval/gsm8k_api_eval.py` 的 API 接口，稍作改造即可生成数据，无需额外计算资源。

---

### 方向二：错误驱动的迭代蒸馏（Error-Driven Iterative Distillation）

**核心思路：** 将评测-训练变成闭环，用 badcases 定向补充训练数据

```
第 0 轮：SFT on 通用数学数据
    ↓
评测 → 收集 badcases（已有 backfill_badcases.py）
    ↓
第 1 轮：Teacher 针对 badcases 生成详细推理 → 补充到训练集（高权重）→ 再次 SFT
    ↓
评测 → 收集新 badcases
    ↓
第 2 轮：重复（2–3 轮即可）
```

**与传统方案的区别：**

| | 传统方案 | 迭代蒸馏 |
|--|------|------|
| 流程结构 | 线性（训练 → 评测，结束） | 闭环（训练 → 评测 → 补数 → 再训练） |
| 数据来源 | 静态公开数据集 | Teacher 针对 Student 弱点动态生成 |
| Teacher 角色 | 仅参与对比评测 | 错误诊断 + 数据生成 |

**实验对比表（作为报告 Results 的核心图表）：**

| 轮次 | 新增数据来源 | GSM8K | BBH |
|------|---------|:-----:|:---:|
| 初始 SFT | NuminaMath 50k | ? | ? |
| 迭代第 1 轮 | badcases → Teacher 生成 ~500 条 | ? | ? |
| 迭代第 2 轮 | 新 badcases → Teacher 生成 ~500 条 | ? | ? |

---

### 方向三：难度感知课程蒸馏（Curriculum Distillation）

**核心思路：** 用 Teacher 的评测结果估计题目难度，给 Student 设计由易到难的学习课程

```
Teacher 对每道题评测 → 按准确率分档
  ├── Teacher 答对 + Student 也答对  →  Easy（价值低）
  ├── Teacher 答对 + Student 答错   →  Hard（蒸馏价值最高）
  └── Teacher 也答错               →  超出蒸馏范围（过滤）
  
Student 训练顺序：Easy（第 0–300 步）→ Medium（300–800 步）→ Hard（800–1800 步）
```

**对比实验设计：**

| 实验组 | 训练顺序 | 预期效果 |
|------|------|------|
| 实验 A（当前方案） | 随机打乱 | 基准线 |
| 实验 B | Easy → Hard 课程 | 预期更好 |
| 实验 C（对照） | Hard → Easy | 预期更差 |

---

### 三方向整合框架

```
Teacher-Adaptive Distillation Pipeline (TADP)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Stage 1: Curriculum SFT
  Teacher 评分 → 难度分档 → 课程训练（Easy→Hard）

Stage 2: Student-Error-Guided DPO
  Student 错误输出 + Teacher 正确输出 → 定制化 DPO pairs

Stage 3: Iterative Refinement（可选，时间充裕时）
  评测 badcases → Teacher 补充数据 → 小步迭代
```

**与现有方案的创新点对比：**

| 维度 | 当前方案 | TADP 扩展方案 |
|------|------|------|
| 数据来源 | 静态公开数据集 | Teacher 动态生成 |
| DPO 数据 | Orca-Math（分布有偏差） | Student 真实错误 + Teacher 纠正 |
| 训练顺序 | 随机打乱 | 难度感知课程 |
| 训练流程 | 线性（训练→评测，结束） | 闭环（训练→评测→补数→再训练） |
| Teacher 模型角色 | 仅对比评测 | 数据生成 + 难度评估 + 错误纠正 |

## 五、核心策略总结

> **诚实报告失败、深入分析原因，比捏造数据掩盖问题的报告得分更高，也更安全。**

- **SFT 是成功的**：BBH +6pp 是真实可信的改善，重点突出
- **DPO 失败是真实的实验发现**：分析 beta 过高、序列长度截断、数据分布不匹配等原因，并提出 Teacher-Guided DPO 作为改进方向
- **三个创新方向均建立在已有代码基础上**，改动量可控，最推荐先实现方向一
